# KAP File Import for KirraScheduler

**Date:** 2026-02-22 14:30 UTC  
**Phase:** 3D Playback — KAP Import Support

## Summary

Implemented `.kap` (Kirra Application Project) file import in KirraScheduler. The KAP format is a ZIP archive containing JSON files, produced by the Kirra App (Kirra2D). This enables importing pit shell surfaces, blast holes, drawings, and charge configurations directly from Kirra into the scheduler's 3D Playback view.

## KAP Format Reference

A `.kap` file is a ZIP (DEFLATE-compressed) containing:
- `manifest.json` — version, project name, content counts
- `surfaces.json` — triangulated surfaces with full vertex geometry (can be 80MB+)
- `holes.json` — blast holes array (Kirra data model)
- `drawings.json` — KAD entities (points, lines, polygons, circles, text)
- `configs.json` — charge configurations
- `products.json` — explosive products
- `charging.json` — per-hole charging data
- `layers.json` — layer organization
- `textures/` folder — OBJ texture blobs
- `images/` folder — GeoTIFF imagery blobs

**Key discovery:** KAP surfaces store triangles in **vertex-per-triangle** format:
```json
{ "vertices": [{"x":..,"y":..,"z":..}, {"x":..,"y":..,"z":..}, {"x":..,"y":..,"z":..}], "minZ":..., "maxZ":... }
```
This differs from the indexed format (`{ a, b, c }` referencing a separate points array).

## Data Flow

1. User drops `.kap` file on Import tab drop zone
2. `FileReader` reads as ArrayBuffer
3. JSZip decompresses the ZIP
4. `manifest.json` parsed for metadata and version check
5. `surfaces.json` parsed — stored in `APP.kirraProjectSurfaces` for 3D rendering
6. `holes.json` parsed — grouped by entityName, matched to existing blasts
7. `drawings.json` parsed — polygons matched to blasts as boundaries
8. `configs.json` parsed — merged into `APP.chargeConfigs`
9. User switches to 3D PLAYBACK tab → surfaces render with elevation gradient

## Test Results

Tested with `KirraProject_20260221_141658.kap` (12.8 MB compressed, ~80 MB surfaces):
- 3 surfaces imported (172,243 + 923 + 832 points; 344,435 + 1,726 + 1,552 triangles)
- 138 blast holes in 1 entity (S4_214_410P)
- 1 polygon drawing
- 9 charge configurations
- 3D pit shell renders correctly with elevation gradient coloring

## Performance Note

The 80MB surfaces.json takes significant time to decompress and parse in the browser (~30-60 seconds). This is expected for production mine site data. A future optimization could use Web Workers for the heavy JSON parsing.
