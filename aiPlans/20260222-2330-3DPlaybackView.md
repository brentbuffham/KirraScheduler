# 3D Playback View — Kirra Scheduler

**Date:** 2026-02-22 23:30
**Status:** PLAN — awaiting approval

## Concept

A 3D animated playback of the blast schedule timeline, showing drill rigs and MPUs moving through the mine site as blasts progress. The user sees a spatial view of the drill plan with equipment physically positioned at their assigned blasts, transitioning day-by-day through the schedule.

## Prerequisites

1. **Spatial data must be loaded** — blasts need polygon boundaries (from DXF import or Kirra project) so drill holes have X/Y/Z positions on the mine bench.
2. **Three.js** — already a dependency in the Kirra App but not in KirraScheduler. Needs adding.
3. **GLTF models** — either imported by the user or bundled as generic defaults.

## Architecture

### New Tab: "3D PLAYBACK"

Add a new tab to the tab bar between CONFORMANCE and EQUIPMENT.

### Files to Create

| File | Purpose |
|------|---------|
| `src/views/playbackView.js` | Main module — Three.js scene setup, animation loop, day stepping |
| `src/three/PlaybackScene.js` | Scene management — camera, lights, ground plane, sky |
| `src/three/EquipmentModels.js` | GLTF model loader + generic fallback meshes |

| `src/three/PlaybackTimeline.js` | Day-by-day timeline scrubber, play/pause/speed controls |
| `src/styles/playback.css` | Overlay controls, timeline scrubber styling |

### Model Strategy

| Equipment | Generic Model Description | GLTF Source |
|-----------|--------------------------|-------------|
| **Drill Rig (PV271)** | Generic Epiroc PV-271 style — tracked base, mast, drill string | User-imported `.glb` or bundled low-poly placeholder |
| **Drill Rig (D65)** | Smaller tracked drill rig — similar silhouette, shorter mast | User-imported `.glb` or bundled low-poly placeholder |
| **MPU** | Volvo FMX dual-axle single-steer 27,500kg emulsion truck | User-imported `.glb` or bundled low-poly placeholder |

**Fallback (no GLTF):** Procedural Three.js geometry:
- Drill: Box body + cylinder mast + simple tracked base
- MPU: Box chassis + cylinder tank + cab geometry

### Scene Layout

1. **Ground plane** — Flat plane at average bench elevation, textured with simple grid or terrain if surface data exists.
2. **Blast polygons** — Extruded outlines on the bench, colour-coded by status:
   - Grey: planned
   - Blue: drilling in progress
   - Orange: loading in progress
   - Red: blast day
   - Green: completed
3. **Drill holes** — Small cylinders at each hole location (only for blasts with spatial data).
4. **Equipment** — GLTF models (or fallbacks) positioned at the active blast polygon centroid.

### Timeline Controls

```
[|<] [<] [>||] [>] [>|]    Day 3 of 42    [2026-02-25]    Speed: [1x] [2x] [5x] [10x]
|=================================>--------------------------------------------|
```

- **Play/Pause** — animate day-by-day at configurable speed
- **Step Forward/Back** — single day increment
- **Jump to Start/End**
- **Speed slider** — 1x, 2x, 5x, 10x real-time
- **Day counter** — shows current simulation day and calendar date
- **Scrubber bar** — click/drag to jump to any day

### Animation Per Day

For each simulation day:
1. Check which blasts are drilling, loading, or blasting.
2. Position drill rigs at their assigned blast polygon centroids.
3. Position MPUs at blasts that are in the loading phase.
4. Animate drill rig mast rotation (simple oscillation to simulate drilling).
5. Colour blast polygons based on phase.
6. Show a small progress bar above each active blast.

### Equipment Movement

When a drill moves from one blast to another (e.g., PV271-01 finishes S4_226_412_V1 drilling and moves to S4_226_411):
- Animate the model along a straight line from old position to new position.
- Could add a "travel day" visual (equipment shown on the path between blasts).

### Camera Controls

- **Orbit** — click+drag to rotate around scene centre
- **Pan** — right-click+drag or middle-click+drag
- **Zoom** — scroll wheel
- **Follow Equipment** — click on a rig to have the camera track it
- **Top-Down** — preset view from above (plan view)
- **Isometric** — preset 45-degree view

### GLTF Import UI

In the EQUIPMENT tab, add an "Import 3D Model" button per equipment type:
- User drops a `.glb` / `.gltf` file
- Stored in memory (and optionally IndexedDB for persistence)
- Used in the playback view instead of the generic fallback

### Data Flow

```
APP.blasts (schedule data)
    |
    +-- blast.polygons[] (XY boundary from DXF import)
    +-- blast.holeTypes[].holes (hole positions if available)
    +-- blast.drillStart / drillDays / loadStart / loadDays / blastDate
    +-- blast.assignedDrills[] / assignedMPU
    |
    v
PlaybackTimeline (builds daily state map)
    |
    v
PlaybackScene (positions equipment, colours polygons)
    |
    v
Three.js WebGL Renderer (canvas in the 3D PLAYBACK tab)
```

## Implementation Phases

### Phase 1: Scene & Static View
- Three.js canvas in new tab
- Render blast polygons as flat coloured outlines
- Camera orbit controls
- No animation yet

### Phase 2: Timeline & Day Stepping
- Timeline scrubber UI
- Day-by-day state calculation
- Polygon colour changes based on phase
- Day counter display

### Phase 3: Equipment Models
- Generic procedural drill/MPU meshes
- Position at blast centroids based on current day
- Simple drilling animation (mast oscillation)

### Phase 4: GLTF Import
- GLTFLoader integration
- Equipment tab "Import 3D Model" button
- Replace procedural meshes with imported GLTF

### Phase 5: Polish
- Equipment movement animation between blasts
- Camera follow mode
- Preset views (top-down, isometric)
- Progress bars above active blasts
- Shadow and lighting refinement
